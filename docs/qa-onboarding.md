# QA Onboarding

Welcome. This guide gets you from zero → first bug report in ~15 minutes.

## 1. Setup

```bash
git clone <repo-url> anthem
cd anthem
bun install
bun run dev
```

The `.env` file is auto-generated by Lovable Cloud — do not edit. App boots at `http://localhost:8080`.

## 2. Run tests

```bash
bunx vitest run          # unit + integration
```

## 3. Get test accounts

Email the security contact in `/SECURITY.md`. We provide:
- 1 admin
- 2 verified users
- 1 unverified user

## 4. Start testing

Use the checklist in [`qa-checklist.md`](./qa-checklist.md). Walk through each section, top to bottom, per release.

## 5. Bug report template

```markdown
**Title**: <short summary>
**Severity**: critical / high / medium / low / cosmetic
**Environment**: preview / production · browser · viewport · OS
**Account used**: <which test account>
**Steps to reproduce**:
1.
2.
3.
**Expected**:
**Actual**:
**Evidence**: screenshot / video / console log / network trace
**Notes**:
```

File via the channel we share (Linear / GitHub Issues / shared sheet).
